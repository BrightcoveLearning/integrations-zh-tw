用戶功能

以下是您可能想要向CMS用戶提供的與Video Cloud相關的功能：

  • 將新視頻添加到Video Cloud
  • 用新版本替換視頻雲視頻
  • 更新視頻的元數據，例如標題，描述和標籤
  • 刪除影片
  • 建立播放清單
  • 更改播放列表中的視頻
  • 刪除播放列表
  • 創建視頻播放器
  • 修改視頻播放器屬性，例如尺寸或樣式
  • 通過插件為視頻播放器添加特殊功能
  • 發佈單個視頻或播放列表
  • 提供有關視頻負載，觀看次數，播放率，參與度等的分析數據。

您可能不希望向最終用戶公開所有這些功能-例如，您可能不想讓他們刪除視頻。將Video Cloud與CMS集成而不是讓用戶直接進入Video Cloud Studio的優勢之一是，您可以選擇通過Brightcove API向用戶公開的功能。

驗證

對於所有Brightcove API請求，身份驗證均基於OAuth2訪問令牌。獲取訪問令牌的過程分為兩個步驟：

  1. 創建具有所需API操作權限的客戶端憑據
  2. 使用客戶端憑據創建臨時訪問令牌以驗證API請求

創建客戶端憑證

創建客戶端憑據是一次性操作，可以通過以下方式執行Video Cloud Studio或者OAuth API。但是，只要您這樣做，client_id和client_secret返回，必須將其保存以請求訪問令牌。

這client_secret僅在創建客戶端憑據時顯示 - 永遠無法再次檢索。如果丟失客戶端機密，則必須創建新的客戶端憑據。

創建訪問令牌

臨時訪問令牌是使用OAuth API。這client_id和client_secret必須經過BASE64編碼並作為Basic授權字符串。

這access_token返回的值通過API調用依次傳遞到Authorization標頭中：

    >Authorization: Bearer your_access_token

訪問令牌的有效期為5分鐘。除非您執行某種批處理操作，該操作將進行數百個連續的API調用，否則只為每個API調用請求一個新的批處理而不是試圖跟踪超時是有意義的。

新增視訊

如果您希望讓用戶從您的 CMS 向 Video Cloud 添加視頻，您可以使用動態攝取 API .我們建議您讓用戶將視頻上傳到您的存儲庫，該存儲庫可以是S3存儲桶，也可以只是面向公眾的服務器。Dynamic Ingest系統可以通過以下兩步過程提取視頻並將其添加到Video Cloud系統。

將視頻對象添加到Video Cloud

第一步是通過在視頻雲系統中創建一個視頻對象，POST向CMS API發送請求：

    https://cms.api.brightcove.com/v1/accounts/:account_id/videos

請求主體將在JSON對像中包含基本視頻屬性-至少，視頻name，但您還可以添加其他元數據，例如description和tags：

    {
    "name": "Woodpecker",
    "description": "A bird that hunts insects inside wood",
    "reference_id": "Bird_Woodpecker.mp4",
    "tags": ["bird", "air", "nature"]
    }
一種reference_id可以添加到任何視頻。參考ID在帳戶中必須唯一。如果您的視頻在您的 CMS 系統中有 id，最好使用該 id 作為 Video Cloud 系統中的參考 id。

攝取視頻

創建視頻對象時，內容管理系統 API將返回一個包含視頻屬性的 JSON 對象。您將提取視頻id從 JSON，並使用它來調用動態攝取 API請求視頻的攝取和轉碼：

    https://ingest.api.brightcove.com/v1/accounts/ACCOUNT_ID/videos/VIDEO_ID/ingest-requests

再次，您將在請求正文中發送JSON，以指定視頻文件的位置：

    {
      "master":{
        "url":"http://learning-services-media.brightcove.com/videos/mp4/Bird_Woodpecker.mp4"
      },
      "profile":"multi-platform-extended-static",
      "capture-images": true
    }
請注意，上面的代碼假設您向用戶提供了一個攝取配置文件選擇器。如果不是，則profile請求正文中應省略字段，在這種情況下，將使用帳戶默認配置文件。這是 Brightcove 建議的做法，除非需要多個攝取配置文件，這並不常見。

這profile這是“摘要配置文件"，用於指定在轉碼過程中應創建哪些演繹版本。在大多數情況下，以下標準配置文件之一應足夠：

動態投放配置文件

  • multi-platform-extended-static
  • multi-platform-standard-static

舊版攝取檔案

  • videocloud-default-v1 (the default)
  • screencast-1280
  • smart-player-transition
  • single-bitrate-high
  • audio-only
  • single-bitrate-standard
  • high-resolution

但是，如果需要，您可以使用來創建其他自定義攝取配置文件提取配置文件API或使用Video Cloud Studio。

管理視訊

這內容管理系統 API允許您取回帳戶的視頻數據。（如上所示，作為視頻攝取過程的一部分，它也用於創建視頻對象。）最基本的要求如下：

    https://cms.api.brightcove.com/v1/accounts/account_id/videos

默認情況下，此請求返回一個包含20個視頻對象的JSON數組，其中包含大量元數據，包括名稱，描述，標籤，自定義字段，創建和最後修改的日期，海報和縮略圖的URL等。

您可以通過向請求中添加以下一個或多個參數來優化請求的結果：

limit
這確定了要返回的視頻對象的數量，並且可以設置為100以上的任何數量-默認值為20
offset
這決定了要跳過的項目數，因此與limit翻閱視頻目錄-默認值為0
sort
這確定了視頻元數據字段，用於對結果進行排序-默認情況下，結果按以下順序排序updated_at（降序，先顯示最近更新的視頻）

看內容管理系統 API概述 - 參數有關這些參數的詳細信息。

搜尋影片

您還可以使用q範圍。您可以按特定字段進行搜索，例如名稱，說明和標籤，以及視頻的日期和狀態：

    https://cms.api.brightcove.com/v1/accounts/account_id/videos?q=tags:sea,mammal

有關詳細信息和所有搜索選項，請參見搜索視頻。

獲取和更新特定視頻

要按ID或參考ID檢索特定視頻，請執行以下操作：

    https://cms.api.brightcove.com/v1/accounts/account_id/videos/id
    or
    https://cms.api.brightcove.com/v1/accounts/account_id/videos/ref:reference_id

一種GET請求返回視頻對象。要對其進行更新，請修改JSON並使用PATCH請求到相同的URL。

在向用戶顯示視頻時，最好在顯示中包含縮略圖，以減少選擇錯誤視頻進行發布或更新的機會。

播放清單

播放列表信息也使用內容管理系統 API與視頻信息的方式大致相同。請注意，Video Cloud支持兩種類別的八種播放列表：

手動（或EXPLICIT）播放列表
包含一組指定的視頻-最多可以包含100個視頻
智能播放列表
根據搜索條件在運行時動態生成-共有七種智能播放列表，它們與列表中視頻的排序方式相對應：
  • ACTIVATEDOLDESTTONEWEST
  • ACTIVATEDNEWESTTOOLDEST
  • ALPHABETICAL
  • PLAYSTOTAL
  • PLAYSTRAILINGWEEK
  • STARTDATEOLDESTTONEWEST
  • STARTDATENEWESTTO_OLDEST
請注意，您可以不是創建播放列表後，將手動播放列表更改為智能播放列表（反之亦然）。

視頻數量的限制可以設置為任意數量，最高為100。

與視頻一樣，您可以使用limit和offset如果該帳戶包含大量的播放列表，則可以分頁瀏覽結果：

    https://cms.api.brightcove.com/v1/accounts/account_id/playlists

返回的播放列表對像數組將包含該播放列表的元數據，包括type對應於上述類型之一。如果類型是EXPLICIT，還會有一個video_ids包含所包含視頻的ID的數組。如果類型是智能播放列表類型之一，則將有一個search包含用於檢索視頻的搜索字符串的屬性，如下所示：

    q=tags:fish,birds

您還可以通過其播放列表來檢索單個播放列表id：

    https://cms.api.brightcove.com/v1/accounts/account_id/playlists/playlist_id

如果您需要檢索播放列表的完整視頻對象（以在頁面上顯示有關視頻的信息），只需添加/videos到該URL：

    https://cms.api.brightcove.com/v1/accounts/account_id/playlists/playlist_id/videos

請注意，對於智能播放列表，請求將返回當前符合搜索條件但可能會更改的視頻。

創建玩家

可以通過以下方式創建Brightcove播放器：播放器管理API。該API可讓您創建播放器，更新其屬性並以URL，iframe標記或要嵌入到頁面中的HTML塊。

每個帳戶最多可以有200個玩家，但是對於用戶來說，擁有絕對需要的玩家數通常會減少混亂。您應該具有用於播放單個視頻或播放列表的單獨的播放器，否則，當它們的樣式不同或通過插件添加了不同的功能時，您僅需要其他播放器。

要創建玩家，您只需製作一個POST向播放器管理API的請求：

    https://players.api.brightcove.com/v2/accounts/account_id/players

在請求的正文中，包括播放器配置 -唯一需要的是name：

    {
    "name": "Single video player for blog posts"
    }

響應將為您提供玩家ID以及多種形式的嵌入代碼：

    {
    "embed_code": "<iframe src='//players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>",
    "embed_in_page": "http://players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/in_page.embed",
    "id": "de055fa4-4f09-45af-8531-419c6794ad04",
    "preview_embed_code": "<iframe src='//preview-players.brightcove.net/v1/accounts/57838016001/players/de055fa4-4f09-45af-8531-419c6794ad04/preview/embeds/default/master/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>",
    "preview_url": "http://preview-players.brightcove.net/v1/accounts/57838016001/players/de055fa4-4f09-45af-8531-419c6794ad04/preview/embeds/default/master/index.html",
    "url": "http://players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html"
    }

要獲得完整的播放器配置，請向/players端點，但添加上面響應中返回的玩家ID：

    https://players.api.brightcove.com/v2/accounts/account_id/players/de055fa4-4f09-45af-8531-419c6794ad04

你可以做一個PATCH請求到同一端點以更新播放器配置。

您會在上面的響應中註意到，preview_embed_code和preview_url。為了允許測試新播放器或播放器更新，在預覽模式下設置了新創建或更新的播放器，以允許您在將更改推送到現有播放器之前看到它。要將變更推向生產，您需要發布提出此要求的玩家：

    https://players.api.brightcove.com/v2/accounts/account_id/players/de055fa4-4f09-45af-8531-419c6794ad04/publish

定制玩家

這Brightcove播放器使用標準的Web技術構建：HTML，CSS和JavaScript。您可以使用相同的技術自定義播放器。可以在發布播放器的頁面上完成此操作，但是最佳做法是通過播放器將自定義設置添加到播放器本身配置，通過PATCH如上一節所述，向播放器管理API請求。

您還可以通過JavaScript向播放器添加其他功能外掛程式，並且有廣泛的播放器API幫助您將代碼與播放器集成在一起。Brightcove提供了許多現成的插件來實現諸如啟用廣告，自定義最終屏幕以及添加覆蓋圖之類的功能。

發佈視訊

在裡面創建玩家部分上面我們看到，當您使用播放器管理API獲取播放器配置對象時，返回的數據包括一個用於將播放器嵌入HTML頁面的iframe標記，如果您想直接將播放器嵌入到HTML頁面中，則返回完整HTML的URL。頁面。

我們建議您盡可能使用 iframe 嵌入，因為這將防止播放器的 CSS 和 JavaScript 與頁面中使用的其他樣式表和腳本之間發生衝突。看這個文件有關差異的詳細信息以及關於在不同情況下使用哪些嵌入的更具體建議。

對於您選擇的任何嵌入，您都需要在嵌入代碼中添加Video Cloud視頻ID或播放列表ID，以向播放器添加內容。iframe嵌入代碼如下所示：

    <iframe
    src='//players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html'
    allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>

在播放器的URL上，您需要附加參數videoId={}video_id，這樣完整的嵌入代碼將如下所示：

    <iframe
    src='//players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html?videoId=4483119716001'
    allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>

如果這是播放列表播放器，則使用參數playlistId={playlist_id}反而。頁內嵌入代碼的修改是相似的。

除非在播放器配置中固定了播放器尺寸，否則您還需要通過在style屬性：

    <iframe
    src='//players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html?videoId=4483119716001'
    allowfullscreen webkitallowfullscreen mozallowfullscreen
    style=width:640px;height:360px;></iframe>

API摘要

以下是可用於與Video Cloud集成的API的摘要。

OAuth API
用於創建客戶端憑據和訪問令牌以訪問其他API。
媒體管理
提取配置文件API
用於創建自定義攝取配置文件，以指定要為添加到Video Cloud的視頻創建的演繹版本
動態提取API
用於將視頻和相關媒體資產添加到Video Cloud
CMS API
用於創建視頻對像以進行攝取，以及管理視頻和播放列表
Brightcove玩家
光明灣球員
播放器包括一個JavaScript API，可在運行時與播放器進行交互
播放器管理API
用於創建和配置播放器，以及獲取播放器嵌入代碼
Analytics API
用於獲取有關視頻效果的分析報告
