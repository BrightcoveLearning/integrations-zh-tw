安裝Brightcove Video Connect for SharePoint

在開始安裝之前，請查看有關先決條件和權限的以下信息：

  • 在用戶可以為SharePoint添加應用程序之前，Farm Administrators組的成員必須配置環境以支持SharePoint應用程序。有關更多信息，請參見為SharePoint 2013配置應用程序環境或者SharePoint 2016 或者 SharePoint 2019。
  • 用戶必須具有管理網站和創建子站點為SharePoint添加應用程序的權限。默認情況下，這些權限僅對具有“完全控制"權限級別或在“網站所有者"組中的用戶可用。
  • 當用戶為SharePoint添加應用程序時，該應用程序會請求其運行所需的權限（例如，訪問“搜索"或創建列表）。沒有這些權限的用戶將被告知他們沒有足夠的權限，因此無法添加該應用程序。用戶可以聯繫站點或服務器場管理員，以查看管理員是否可以添加該應用程序。
  • 使用系統帳戶登錄到站點的用戶無法安裝應用程序。系統帳戶無法導入應用程序許可證，因為這可能會導致性能問題。

將Brightcove Video Connect應用添加到應用目錄

設置應用目錄

要安裝Brightcove連接器，第一步是在SharePoint中設置應用程序目錄。使用下面的參考信息完成此步驟。

  • 在SharePoint 2013中管理應用程序目錄
  • 在SharePoint 2016中管理應用程序目錄
  • 在SharePoint 2019中管理應用程序目錄

將Brightcove Video Connect應用添加到目錄

設置應用目錄後，請按照以下步驟操作：

  1. 驗證執行此過程的用戶帳戶是否是“應用程序目錄"的“網站所有者"或“設計者"組的成員。
  2. 在應用目錄網站上，點擊適用於SharePoint的應用列表。
  3. 在適用於SharePoint的應用頁面上，單擊新物品。
  4. 在裡面選擇一個文件框，點擊瀏覽，然後找到包含 BrightcoveConnectorApp 文件。
  5. 選擇應用程序，然後單擊打開。
  6. 點擊好的上傳應用程序。
  7. 在“項目詳細信息"框中，提供應用程序的名稱，標題，簡短描述，圖標URL和其他設置。確保已啟用複選框處於選中狀態，以便用戶可以在其站點中看到該應用程序。您可以選中“特色功能"複選框以在“精選應用目錄的內容視圖。
  8. 點擊保存。

在站點中安裝Brightcove Video Connect應用程序

請按照下面列出的步驟在站點中安裝Brightcove連接器：

  1. 驗證執行此過程的用戶帳戶是“網站所有者"組的成員。
  2. 可以通過以下兩種方式之一將應用程序添加到站點：
    1. 在任何頁面上，從設置菜單選擇添加應用選項。
    2. 從網站內容使用添加應用選項。
  3. 接下來，單擊“來自您的組織”，然後單擊BrightcoveConnector應用程序
  4. 在“授予權限”對話框中，單擊“信任”。
  5. 此時， Brightcove連接器添加了SharePoint應用程序，該應用程序顯示在“網站內容"列表的“應用程序"部分中。現在可以在站點中使用它。

為Brightcove Video Connect App設置默認列表

在使用Brightcove連接器之前，需要按照以下步驟配置一些必需的列表：

  • 當應用程序首次實例化時，或者如果列表不存在於本地站點中，那麼用戶將看到最多 3 個選項以允許在站點內提供列表。

  • 單擊時，每個按鈕都會顯示一條消息：
  • 單擊“創建設置列表”時，除了創建 Brightcove 設置列表外，還會顯示編輯此列表的提示，以確保配置了 Brightcove API 支持所需的代理 URL。
  • 單擊此鏈接將打開Brightcove設置列表，以立即編輯代理服務器訪問信息。標題必須設置為“代理"。ProxyURL特定於您將用來訪問Brightcove API的代理服務器的位置。看使用代理服務器進行 Brightcove API 通信有關使用代理服務器的更多詳細信息。
  • 完成列表配置將在站點中創建以下列表：
    • 帳戶-代表帳戶對象的自定義列表。
    • 視頻-代表視頻對象的自定義列表。
    • 設置-列表以存儲代理服務器的詳細信息。

安裝SharePoint Online的播放器解決方案包

Brightcove Video Connect軟件包中包含的Brightcove Player解決方案軟件包使用戶可以將Brightcove視頻播放器嵌入SharePoint網站頁面上。Brightcove Player解決方案包使用了現成的SharePoint控件，這些控件與SharePoint Online和SharePoint Server兼容，因此可以同時使用。Brightcove Video Connect軟件包中還提供了用於嵌入視頻的Player Web Part沙箱解決方案，但只能與內部部署的SharePoint Server一起使用，因為Microsoft託管的SharePoint Online和SharePoint Server具有禁止沙箱解決方案的安全限制；這個解決方案是 在下一節中介紹。

要將Brightcove播放器嵌入SharePoint Online中，此處描述的解決方案包允許用戶選擇視頻或播放列表播放器，然後 生成了播放器嵌入代碼，供用戶複製/粘貼到“SharePoint嵌入代碼"對話框中。

注意-此部分需要BrightcovePlayer.wsp解決方案文件。

  1. 在安裝 Brightcove 應用程序的站點集合的根站點中，導航到站點設置 ->網站集管理 -> HTML 字段安全。
  2. 將players.brightcove.net 添加為允許的iframe 域
  3. 在安裝 Brightcove 應用程序的站點集合的根站點中，導航到站點設置 ->網頁設計師畫廊 ->解決方案。
  4. 將“BrightcovePlayer.wsp”文件上傳到解決方案庫並激活該文件。
  5. 導航到站點設置 ->網站操作 ->管理站點功能。
  6. 激活“Brightcove Player”功能。
  7. 驗證是否在安裝期間創建了 BrightcoveConnectorResources 文檔庫。
  8. 如果沒有，請創建一個名為 BrightcoveConnectorResources 的新文檔庫
  9. 並將 BrightcovePlayer.css 和 BrightcovePlayer.js 上傳到該文檔庫。

安裝代理服務器以進行Brightcove API通信

由於連接器是客戶端JavaScript應用程序，因此需要代理服務器來代理與Brightcove API的身份驗證通信。有關更多信息，請參見Brightcove API文檔。

Brightcove Video Connect軟件包中提供了一個.net代理應用程序。代理服務器應部署在可訪問Web的位置，例如Microsoft Azure上。出於安全原因，代理被設計為僅允許來自與代理一起安裝的web.config文件中包含的網站的連接。web.config文件中的brightcove：origins項應包括以下內容的SharePoint網站URL： Brightcove連接器應用以及網站所在的網站的網址BrightcovePlayer組件將執行。請參見下面的示例：

<add key="brightcove:origins" value="https://brightcovecms-a3da847a1c9e33.sharepoint.com, https://brightcovecms.sharepoint.com" />

多個站點可以以逗號分隔的字符串形式列出。

啟動Brightcove Video Connect for SharePoint

要啟動該應用程序，用戶可以單擊Brightcove連接器網站內容列表中的應用程序，或者單擊菜單鏈接（如果由SharePoint管理員配置）。

無論哪種情況，都會將用戶帶到連接器應用程序的主頁。客戶端可以將該頁面進一步配置為包括其他公司特定的消息傳遞，其他鏈接等。如安裝部分所述，根據用戶的組成員身份，他們可能會或可能不會看到“帳戶管理"和“添加視頻"選項卡。

連接器左欄中的主導航始終存在，並允許用戶導航他們有權訪問的所有部分（請參閱上面有關權限）。下面是連接器每個組件用途的簡要說明：

  • 家 -首次進入連接器時的登錄頁面。
  • 帳戶管理 -提供用於將Brightcove視頻雲帳戶信息輸入SharePoint的界面。
  • 新增影片 -將新視頻上傳到Video Cloud。在SharePoint中也會創建一個視頻對象（實際的視頻文件除外）。
  • 管理影片 -瀏覽和編輯現有視頻，以及將視頻從“視頻雲"導入SharePoint。
  • 管理播放清單 -創建新的播放列表，以及瀏覽和管理現有的播放列表。

也有從Brightcove添加視頻和編輯視頻執行這些操作時將顯示在主導航中的組件。

鏈接到視頻雲帳戶

Connector的“帳戶管理"部分用於輸入將在站點中訪問的每個鏈接的Brightcove Video Cloud帳戶的信息。無法從SharePoint創建Brightcove帳戶，必須先在Video Cloud中創建它們。在訪問Brightcove API與Video Cloud進行通信時，必須將Brightcove帳戶信息輸入到SharePoint中。它還提供了分配帳戶特定訪問權限的方法。對於多帳戶企業客戶端，它可以控制哪些用戶可以訪問每個帳戶，以及他們是否只能瀏覽或上傳視頻。

帳戶管理登陸頁面

帳戶管理登錄頁面顯示當前已配置帳戶的列表，以及為作者和查看者分配的組。

  1. 帳戶名稱 -當前配置（或至少創建）的帳戶。
  2. 作者組 -SharePoint組為帳戶分配了“作者"權限。
  3. 觀眾組 -SharePoint組為該帳戶分配了“查看者"權限。
  4. 新增帳號 -打開帳戶創建屏幕以設置新帳戶。

創建/編輯帳戶

無論是創建新帳戶還是編輯現有帳戶，過程和選項都是相同的。首先需要從Video Cloud帳戶獲取所有帳戶屬性（“作者"和“查看者"組除外）。以下是每個帳戶所需的屬性。

  1. 帳戶名稱 -帳戶名稱。此名稱僅在SharePoint中使用，因此不必與Video Cloud中的名稱相同。
  2. DefaultVideoPlayerId -設置視頻的默認播放器。
  3. DefaultPlaylistPlayerId -設置默認播放列表播放器。
  4. 帳戶ID -來自Video Cloud的Brightcove帳戶ID。
  5. 客戶編號 -Brightcove API身份驗證客戶端ID。
  6. 客戶機密 -Brightcove API身份驗證客戶端密鑰。
  7. 臨時存放地點 -在將視頻和文本軌道上傳到Video Cloud時要使用的臨時存儲位置。下拉菜單允許選擇Amazon Web Services（AWS）或Dropbox。
  8. AWS訪問密鑰ID -如果使用AWS，則在此處輸入訪問密鑰ID。
  9. AWS秘密訪問密鑰 -如果使用AWS，則在此處輸入秘密訪問密鑰。
  10. AWS Bucket名稱 -如果使用AWS，則在此處輸入AWS Bucket Name。
  11. Dropbox帳戶ID -如果使用Dropbox，則在此處輸入Dropbox帳戶ID。
  12. Dropbox訪問令牌 -如果使用Dropbox，則在此處輸入Dropbox訪問令牌。
  13. 自定義字段 -用於映射在Video Cloud中為此帳戶創建的自定義字段。見章節自定義字段映射想要查詢更多的信息。
  14. 作者組，查看者組 -帳戶的SharePoint側權限定義組。
    1. 作者組（在本例中為SharePoint“所有者"組）的成員可以在帳戶中執行需要寫令牌的功能，例如添加視頻，編輯和更新現有視頻，創建/編輯播放列表等。 。他們還可以執行所有閱讀類型功能，例如瀏覽和搜索視頻和播放列表。
    2. 查看者組的成員（在本例中為SharePoint“訪問者"組）僅被授權使用讀取類型的功能，例如瀏覽和搜索視頻和播放列表。

要設置Brightcove帳戶ID，請選擇“帳戶ID"按鈕，然後按照以下模式輸入在上一步中保存的Brightcove發布者ID：

要設置Brightcove客戶ID，請選擇“客戶ID"按鈕，然後按照以下模式輸入您在上一步中保存的客戶ID：

要設置Brightcove客戶端機密，請選擇“客戶端機密"按鈕，然後按照以下模式輸入在上一步中保存的客戶端機密：

  1. 價值 -輸入屬性的值。可以使用視頻雲數據中的剪切和粘貼（如果有）輸入值。

定義 視訊雲端 播放器

要設置應在SharePoint中用作默認單視頻播放器的播放器ID，請選擇“DefaultVideoPlayerId"按鈕，按照以下模式從Brightcove輸入播放器ID：

  1. 價值 -輸入屬性的值。可以使用視頻雲數據中的剪切和粘貼（如果可用）輸入該值。

新增文字軌

文字軌可以在第一次上傳或後續更新時添加到任何視頻中。一個視頻可以有一個或多個與其相關聯的文本軌道。用戶可以選擇要添加到視頻的文本軌道類型（文件上傳，URL 上傳或遠程 URL）。如果選取「無」，區段將不會顯示任何欄位。

重要的：單一影片不支援檔案上傳、網址上傳或遠端 URL 的混合。每個視頻只能與一種類型的文本軌道相關聯。

檔案上傳

當文字追蹤檔案選取「檔案上傳」時，會顯示下列欄位：

  1. 選擇文字軌道的類型 -允許選擇“文件上載"，“URL上載"或“遠程URL"。
  2. 文件選擇器 -選擇文本跟踪文件。
  3. 文字跟踪屬性 -設置與文本軌道關聯的屬性。點擊「添加」將文本軌道添加到下次上傳/更新視頻數據。
    1. 語 -文本軌道語言的兩個字母的代碼（有效的BCP 47語言標記），例如英語的“en"。
    2. 標籤 -用戶可以看到的曲目標籤，例如在菜單中列出了可用於字幕的不同語言。
    3. 種類 -列出的五種受支持的曲目類型之一
  4. 添加/排隊的文本軌道列表 -顯示排隊等待下一次保存的現有文本軌道和新文本軌道的列表。按一下 [移除] 連結並儲存影片，即可刪除文字軌。

  • 遠端 URL-輸入文字追蹤檔案所在的公用 URL。網址必須少於 250 個字元。字幕將從遠程 URL 加載。
  1. 選擇文字軌道的類型 -允許選擇“文件上載"，“URL上載"或“遠程URL"。

    1. 語 -文本軌道語言的兩個字母的代碼（有效的BCP 47語言標記），例如英語的“en"。
    2. 標籤 -將顯示給用戶的曲目標籤，例如在列出字幕可用語言的菜單中。

  4. 添加/排隊的文本軌道列表 -顯示排隊等待下一次保存的現有文本軌道和新文本軌道的列表。按一下 [移除] 連結並儲存影片，即可刪除文字軌。

  1. 搜尋欄位 -用戶可以在此處輸入搜索詞，然後單擊放大鏡（或點擊返回）以執行搜索。
  2. 僅顯示SharePoint或僅顯示視頻雲 -將視頻篩選為僅在SharePoint中錄製的視頻或僅駐留在Video Cloud中且從未將其數據導入SharePoint的視頻。
  3. 排序/排序過濾器 -用於根據列出的參數對任何結果進行排序和排序，以幫助更快地找到視頻。

添加，導入和刪除播放列表

Connector 的「管理播放清單」區段可用來建立新的播放清單，以及瀏覽和管理現有的播放清單，包括編輯和更新。與影片不同的是，播放清單資訊不會儲存在 SharePoint 中，而是在每次瀏覽帳戶時從 Video Cloud 擷取。基於這個原因，通過 API 播放列表數據的可用性有一些固有的延遲，這將在以下幾節中調用。

管理播放清單登陸頁面

「管理播放清單登陸頁面」會顯示可用來瀏覽播放清單或在可用帳戶中建立新播放清單的欄位清單：

  1. 選擇帳號 -該下拉列表將填充有該用戶有權訪問的所有帳戶；這是通過“帳戶組"分配啟用的。使用者必須選取帳號才能檢視任何播放清單。
  2. 播放列表顯示列 -播放列表數據將顯示在這些列中。按一下欄標題，即可依任何欄排序結果。
  3. 新增播放清單 -點擊此按鈕可以在所選帳戶中創建新的播放列表。

選擇一個帳戶後，將顯示該帳戶的所有播放列表。

  1. 選擇帳號 -該下拉列表將填充有該用戶有權訪問的所有帳戶；這是通過“帳戶組"分配啟用的。使用者必須選取帳號才能檢視任何播放清單。
  2. 播放列表顯示列 -通過單擊列標題，可以按任何列對列表進行排序。如果列表過長，這可以幫助找到播放列表。
  3. 播放清單清單 -每個播放列表列表包含以下內容：
    1. 播放清單名稱-按一下名稱即可開啟播放清單以編輯其資料。
    2. 播放清單中有效的視訊 ID 清單。
    3. 簡短描述。
    4. 播放列表類型- 手動或智能播放列表。
    5. 播放清單的 ID (影片雲端播放清單 ID)。
    6. 「編輯播放清單」連結。此連結 (或按一下「名稱」) 用來開啟播放清單以進行編輯或檢視其詳細資料。
  4. 新增播放清單 -點擊此按鈕可以在所選帳戶中創建新的播放列表。

將視訊新增至 SharePoint 網站

SharePoint連接器允許內容作者以多種方式將媒體插入其網站。本章介紹了可用的選項。

用於內部部署SharePoint的Brightcove Player Web部件

Brightcove Player Web部件沙箱解決方案生成一個Web部件，該Web部件放置在內部或外部網站的“站點"頁面上，並播放Brightcove視頻。SharePoint Online限制了用戶添加自定義Web部件的能力，因此，這種用於插入視頻的機制僅適用於本地SharePoint安裝。

Web部件在管理控制台中也增加了一些相同的功能，包括“帳戶選擇"和具有排序和排序功能的高級搜索，以更快地找到視頻和播放列表。

若要查看Player Web Part選項，SharePoint頁面需要處於“編輯"模式（1），它將公開用於編輯Web Part的選項（2）。Web部件特定的選項可在Web部件編輯器的“Brightcove配置"部分（3）中找到。

展開“Brightcove配置"部分將顯示Player Web部件沙箱解決方案的更新功能。

選擇要在Web部件中播放的視頻

  1. Brightcove配置 -Web部件編輯器的此部分提供對用於配置Web部件和選擇要播放的視頻/播放列表的參數的訪問。
  2. 播放器寬度，播放器高度 -設置播放器的尺寸。
  3. 玩家編號 -指定要使用的播放器的ID。重要的是，如果選擇的播放器是多帳戶環境，則與所選視頻來自同一帳戶。播放器沒有“選擇器"，因此必須手動輸入來自Video Cloud的ID。
  4. 影片編號 -使用視頻選擇器填充“視頻ID"字段。如果用戶單擊字段右側的省略號框，則會顯示“選擇器"窗口。
  5. 選擇器窗口
    • 選擇帳號 -顯示當前用戶有權訪問的帳戶。
    • 可用影片 -將視頻結果過濾為包含該術語的結果。搜索名稱，標籤和描述信息以及SharePoint元數據字段（如果已定義）。一次只能在一個帳戶中進行搜索；不支持跨帳戶搜索。
    • 排序和排序
    • 視頻列表窗口 -單擊視頻將其選中以進行播放。該列表僅限顯示30個視頻，因此，在首次選擇一個帳戶時，可能不會顯示該帳戶中的所有視頻。使用搜索功能可以優化列表以找到所需的視頻。
    • 確定按鈕 -提交選擇，並在視頻ID字段中填充適當的ID值。

設置播放器ID和視頻ID之後，就可以配置Web部件並開始播放了！

選擇要在Web部件中播放的播放列表

選擇要播放的播放列表的過程幾乎與選擇視頻相同。

  1. Brightcove配置 -Web部件編輯器的此部分提供對用於配置Web部件和選擇要播放的視頻/播放列表的參數的訪問。
  2. 播放器寬度，播放器高度 -設置播放器的尺寸。
  3. 玩家編號 -指定要使用的播放列表播放器的ID。如果所選播放器是多帳戶環境，並且配置為顯示播放列表，則所選播放器與所選視頻來自同一帳戶，這一點很重要。播放器沒有“選擇器"，因此必須手動輸入來自Video Cloud的ID。
  4. 播放清單編號 -使用視頻選擇器填充視頻ID。如果用戶單擊字段右側的省略號框，則會顯示“選擇器"窗口。
  5. 選擇器窗口
    • 選擇帳號 -顯示當前用戶有權訪問的帳戶。
    • 排序和排序
    • 播放列表列表窗口 -點擊播放列表將其選中進行播放。該列表僅限於顯示30個播放列表，因此，在首次選擇一個帳戶時，可能不會顯示該帳戶中的所有播放列表。使用搜索功能可以優化列表以找到所需的播放列表。
    • 確定按鈕 -提交選擇，並使用適當的ID值填充“播放列表ID"字段。

設置播放列表ID和視頻ID後，就可以配置Web部件並開始播放了！
