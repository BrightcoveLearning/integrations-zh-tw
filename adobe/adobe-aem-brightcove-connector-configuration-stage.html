Adobe-AEM-Brightcove 連接器：配置

本主題涵蓋Adobe Experience Manager（AEM）的Brightcove連接器的配置。

簡介

Abe-AEM-BrightCove 連接器可讓您管理 AEM 中的 Brightcove 視訊雲端視訊和播放器，並輕鬆將影片嵌入 AEM 頁面中。

筆記：
  • 如果您使用的是 Adobe Experience Cloud，請務必查看Adobe 體驗雲附加配置要求部分/
  • 本指南使用 AEM 6.3。該連接器還可以與AEM 6.2、6.4和6.5一起使用，並且步驟類似。

配置步驟

以下各節將引導您完成使用經典界面配置Adobe-AEM-Brightcove連接器的步驟。請注意，在完成這些步驟之前，必須先安裝連接器-請參閱Adobe-AEM-Brightcove連接器：安裝

輸入帳戶信息

現在，您可以使用視頻雲帳戶信息來配置AEM-Brightcove連接器。

  1. 打開AEM Web控制台配置管理器（http：// localhost：4502 / system / console / configMgr）
  2. 搜尋Brightcove服務查找服務：
  3. 點擊+登錄以添加新配置：
  4. 在對話框中，輸入如下所示的值。看管理API身份驗證憑據有關獲取客戶端ID和客戶端機密的說明。這裡需要的權限是：
    • 以下是配置表單的建議值：
      • 帳戶別名：輸入要在連接器中顯示的帳戶的名稱（必填）
      • 帳戶ID：輸入您的Brightcove發布者ID（必填）
      • 客戶編號：從Brightcove API身份驗證頁面輸入您的Brightcove客戶端ID（必填）
      • 客戶機密：輸入您的Brightcove客戶端秘密API（必填）
      • 允許的團體：請指定將訪問連接器的組（必填），確保該組是您所包含的組。
      • 玩家商店路徑：保留默認值或使用文件夾 此Brightcove帳戶的玩家選擇（必填）
      • 默認視頻播放器ID：保留默認值或選擇其他玩家ID（必填）
      • 默認視頻播放器密鑰：已棄用，保留為空白或使用先前連接器的值
      • 默認播放列表播放器ID：如果您打算使用播放列表，則輸入為播放列表啟用的播放器ID（必填）
      • 默認播放列表播放器鍵：已棄用，保留為空白或使用先前連接器的值
      • 代理服務器：留空或輸入代理（如果您以前使用過代理）
      • 大壩整合路徑：保留此Brightcove帳戶的視頻資產的默認值或使用您選擇的文件夾（必填）
      • 攝取資料：指定用於上傳視頻的Brightcove攝取配置文件，空白值將使用默認配置文件，如果空白/默認值不起作用，請請求Brightcove支持以確保在視頻雲後端中定義了默認配置文件
  5. 測試您的配置。 瀏覽到http：// localhost：4502 / brightcove / admin，然後選擇右上角下拉菜單中剛剛添加的Brightcove帳戶。 如果未顯示任何視頻，則可能是由於以下原因之一：
    1. 您的API憑證 帳戶ID，客戶編號， 和客戶機密 可能是錯誤地在連接器配置中輸入的
    2. 您的API憑據未在Brightcove中正確配置
    3. 當前登錄的用戶不是以下用戶之一的成員允許的團體 在連接器配置中
    4. 您的網絡配置阻止了API對Brightcove的雲服務器的訪問。看“調試網絡和API連接問題"想要查詢更多的信息。
  6. 如果要使用多個Video Cloud帳戶，請對每個帳戶重複前面的步驟。

Adobe 體驗雲

在 AEM Cloud 上使用 Brightcove 連接器需要進行一些配置更改。

  1. 在您的部署中包含一個 OSGI 配置來定義您的帳戶信息。這與以前的 Brightcove 版本具有所有相同的信息，但必須部署在您的 ui.config 項目或類似項目中。
  2. 如果您希望在發佈時將資產更新與 Brightcove 同步，請將復制偵聽器的 OSGI 配置作為部署的一部分。這取代了先前連接器版本中包含的已棄用的自定義 Replication Agent。
  3. 請務必為 DAM 中的每個 Brightcove 帳戶文件夾啟用 DAM 設置上次修改日期工作流啟動器。默認情況下，在最新的 Cloud 實例上禁用此功能。
  4. 與以前的版本類似，連接器帶有一個（可選的）自動元數據模式選項卡。您必須選擇此架構並將其手動應用到 Brightcove 文件夾，而不是自動應用。
  5. 此更新需要 Java 11。如有必要，更新您的版本。
  6. 您需要將此依賴包嵌入到他們的 Maven 項目中。包需要作為供應商提供的 pom.xml 中的依賴項引用。
  7. 為此，您可以在 pom.xml 中使用本地文件存儲庫，如下例所示： 
    <repositories> <repository> <id>my-local-repo</id> <url>file://${project.basedir}/my-repo</url> </repository> </repositories>
  8. 然後您可以使用以下命令安裝連接器包： 
    mvn org.apache.maven.plugins:maven-install-plugin:2.3.1:install-file \ -Dfile=<path-to-file> -DgroupId=com.coresecure \ -DartifactId=brightcove.connector -Dversion=1.0.0 \ -Dpackaging=jar -DlocalRepositoryPath=<path-defined-above>
  9. 最後，您必須通過 pom.xml 的依賴項部分引用本地安裝的包： 
    <dependency> <groupId>com.coresecure</groupId> <artifactId>brightcove.connector</artifactId> <version>1.0.0</version> </dependency>
