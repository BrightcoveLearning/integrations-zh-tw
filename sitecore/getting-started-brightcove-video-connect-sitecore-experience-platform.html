安裝 Brightcove Video Connect 視頻雲網站核心內容管理系統

本主題涵蓋 Brightcove 視訊連線模組的安裝程序。

 

安裝視頻雲網站核心媒體框架 2.1

Brightcove 視頻連接需要視頻雲網站核心要先安裝媒體框架。這視頻雲網站核心使用 Media Framework 安裝視頻雲網站核心安裝嚮導。依照安裝精靈顯示的指示進行。

從工作流程發行頁面事件

這視頻雲網站核心媒體框架包括許多頁面事件，必須從工作流中釋放並發布才能使用它們。

  1. 打開視頻雲網站核心工作箱
  2. 選取分析工作流程
  3. 部署下列頁面事件：
    • 播放已變更
    • 播放完成
    • 播放錯誤
    • 開始播放
  4. 發佈頁面事件

安裝 Brightcove Video Connect 視頻雲網站核心體驗平台

Brightcove 視頻連接視頻雲網站核心安裝使用視頻雲網站核心安裝嚮導。依照安裝精靈顯示的指示進行。這視頻雲網站核心插件可通過上表中的鏈接下載。此 zip 檔案包含來源和外掛程式套件。 先解壓，然後 將內部 zip 包上傳到您的視頻雲網站核心實例。

報表資料庫組態

為了使視頻雲網站核心要使媒體框架報告工作，必須將新的事實表添加到體驗數據庫 (xDB) 報告數據庫中。將創建這個新表的腳本在 SDN 上可用。您必須在 XDB 報告資料庫上執行此指令碼。此步驟應由您的資料庫管理員完成。

注意：可以安裝多個視頻雲網站核心媒體框架連接器。無論您已安裝多少個不同的連接器，這個指令碼只需要每個報告資料庫執行一次。

多伺服器組態

視頻雲網站核心媒體框架可以安裝在多服務器內容創作環境中。如果你的視頻雲網站核心環境是這樣配置的，需要配置如下：

內容管理伺服器

如果您使用排程器來同步處理媒體，則只有一個內容管理 (CM) 伺服器需要設定排程器。從其他 CM 伺服器上的設定檔中移除排定的工作。這視頻雲網站核心事件隊列將處理保持其他 CM 服務器同步。

注意：在多個 CM 伺服器上設定排程器不會造成問題，但這樣做並沒有好處。

內容傳遞伺服器

在每個內容傳遞 (CD) 伺服器上，從指定的設定檔中移除下列索引。只有在 CM 環境中才需要此索引。

檔案： Video Cloud Sitecore .MediaFramework.Services.Brightcove.config
索引名稱： MediaFramework_Brightcove_Index
 

設定 CD 伺服器時，您可能會發現從 CM 伺服器複製組態檔很方便。如果您這麼做，且已新增排程器來同步處理媒體，您應該從 CD 伺服器上的設定檔中移除排定的工作。媒體同步處理不應該在 CD 伺服器上執行。它應該在一個 CM 伺服器上執行。發佈是用來讓媒體項目可以在 CD 伺服器上使用。

鏈接視頻雲帳戶

本主題介紹如何在視頻雲網站核心對應於 Brightcove 視頻雲帳戶。

 

從布萊特灣取得帳戶憑證

為了讓媒體框架連接到 Brightcove 視頻雲，必須將來自 Brightcove 的帳戶憑據添加到視頻雲網站核心 .本節說明如何在 Brightve 中找出帳戶認證。以下部分介紹瞭如何將這些憑據添加到視頻雲網站核心 .

  1. 登錄 Brightcove 視頻雲 .請注意「發行者識別碼」值。您需要將其輸入視頻雲網站核心在下一節中。
  2. 瀏覽至「管理員」。
  3. 導覽至 API 驗證。
  4. 註冊新的應用程式。
  5. 為應用程式命名、設定帳戶存取權，以及啟用 API。
  6. 請注意用戶端識別碼和用戶端密碼。您需要將它們輸入視頻雲網站核心在下一節中。

將 Brightcove 帳戶添加到視頻雲網站核心

本節介紹如何將 Brightcove 帳戶添加到視頻雲網站核心 .

  1. 登錄視頻雲網站核心桌面。
  2. 開啟 [媒體庫]。
  3. 導覽至 [媒體庫] > [媒體架構] > [帳戶]。
  4. 新增新的布萊特灣帳號項目。
  5. 輸入上一節所述的「發行者識別碼」、「用戶端識別碼」和「用戶端密碼」值。
  6. 儲存料件。

定義視頻雲球員

需要播放器才能顯示單一影片或影片播放清單。播放器定義了什麼樣的功能可用於誰想要觀看視頻的訪問者。視頻雲網站核心 Media Framework 允許內容作者添加新播放器、編輯現有播放器和刪除現有播放器。

 

新增播放程式

本節介紹如何手動創建視頻雲網站核心與玩家對應的物品。

  1. 在布賴特科夫視頻雲，導航到玩家部分。
  2. 選擇您要在其中創建的播放器視頻雲網站核心 .請注意播放器 ID 值。您需要將其輸入視頻雲網站核心在下一節中。
  3. 在視頻雲網站核心 , 在媒體庫中，選擇 Brightcove 帳戶項目。
  4. 選取「帳戶」項目。
  5. 選取 [玩家] 項目。
  6. 新增新的布萊特灣影片播放器物品。系統會提示您輸入玩家 ID。輸入 Brightcove 的值視頻雲 .

設定預設播放器

為了預覽視頻和播放列表視頻雲網站核心，必須指定默認播放器。本節說明如何設定預設播放器。

  1. 在視頻雲網站核心 , 在媒體庫中，選擇 Brightcove 帳戶項目。
  2. 選取「帳戶」項目。
  3. 選取「設定」項目。
  4. 設定「預設視訊播放程式」欄位和「預設播放清單播放程式」欄位的值。
  5. 儲存項目

編輯播放程式

播放器物品的編輯方式與其他物品一樣視頻雲網站核心項目。

刪除播放程式

播放器項目像其他任何項目一樣被刪除視頻雲網站核心項目。

注意：刪除玩家視頻雲網站核心不會導致玩家從 Brightcove 中刪除視頻雲 .

匯入、新增、預覽和刪除視訊

視頻雲網站核心媒體框架允許內容作者從他的本地機器上傳視頻，以及編輯和刪除現有視頻。您可以定義新標籤來組織視訊內容。

 

匯入視訊

一種視頻雲網站核心內容作者或管理員可以手動觸發媒體導入過程。

導入視頻時，視頻雲網站核心將反映導入視頻的當天而不是其中的上傳日期視頻雲 .如果您要移動以前存在的視頻雲網站核心實施，您可以遷移以前的內容，連接器將識別這些視頻。

Brightcove 視訊連線外掛程式包含定期自動同步處理媒體的代理程式。代理程式定義在下列位置：

設定檔： Video Cloud Sitecore .MediaFramework.Services.Brightcove.config
代理程式名稱： MediaFramework_Import_Brightcove
  1. 通過右鍵單擊頂部菜單並選擇媒體框架顯示媒體框架功能區。
  2. 按一下媒體架構功能區。
  3. 按一下「全部匯入」按鈕。
  4. 匯入程序執行時，會出現狀態方塊。當狀態方塊消失時，匯入程序已完成。
  5. 若要確認媒體已匯入，請按一下 [搜尋] 索引標籤。
  6. 執行搜尋。
  7. 篩選視訊。
  8. 顯示搜尋結果。

加入視訊

視頻雲網站核心媒體框架允許內容作者添加新視頻。

  1. 打開視頻雲網站核心媒體庫。
  2. 瀏覽至您要建立播放清單的影片帳戶。
  3. 瀏覽至「媒體內容」項目。
  4. 點擊布萊特灣視頻。
  5. 按一下「新增檔案」。
  6. 選擇您要從本機電腦上傳的檔案。
  7. 按一下「開始」。
  8. 影片上傳後，關閉上傳者視窗。

定義標籤

視頻雲網站核心媒體框架允許內容作者添加新標籤。

  1. 打開視頻雲網站核心媒體庫。
  2. 瀏覽至您要為其建立標籤之影片的帳戶。
  3. 瀏覽至「標籤」項目。
  4. 按一下「布萊特灣標籤」。
  5. 輸入料號名稱。這將成為標籤。

編輯視訊和標籤

視頻和標籤項目的編輯方式與其他項目一樣視頻雲網站核心項目。

刪除視訊和標籤

視頻和標籤項目像其他任何項目一樣被刪除視頻雲網站核心項目。

搜尋影片和播放清單

您可以在影片和播放清單的名稱和說明中搜尋文字字串。您也可以在自訂欄位中搜尋文字字串來尋找視訊。

 

搜尋

按一下 [搜尋] 索引標籤以搜尋媒體。在名稱或描述欄位中輸入要搜尋的文字。

按一下放大鏡以執行搜尋。

篩選

搜尋結果可依類型 (例如影片或播放清單)、日期和其他屬性篩選。

顯示搜尋結果。

匯入、新增、預覽及刪除播放清單

播放清單是單一播放器中的視訊集合。視頻雲網站核心媒體框架允許內容作者添加新播放列表、編輯現有播放列表和刪除現有播放列表。

導入播放列表時，視頻雲網站核心將反映播放列表導入的當天而不是其中的創建日期視頻雲 .如果您要移動以前存在的視頻雲網站核心實施，您可以遷移以前的內容，連接器將識別這些視頻。
 

新增播放清單

視頻雲網站核心媒體框架允許內容作者添加新的播放列表。

  1. 打開視頻雲網站核心媒體庫。
  2. 瀏覽至您要建立播放清單之影片的帳戶。
  3. 瀏覽至「媒體內容」項目。
  4. 新增新的布萊特灣播放清單項目。
  5. 在「名稱」欄位中，輸入視訊的描述性名稱。
  6. 在「視訊 ID」欄位中，選取您要包含在播放清單中的影片。

編輯播放清單

播放列表項目的編輯方式與其他任何項目一樣視頻雲網站核心項目。

刪除播放清單

播放列表項目像其他任何項目一樣被刪除視頻雲網站核心項目。

將視頻添加到視頻雲網站核心網站

視頻雲網站核心媒體框架允許內容作者以多種方式將媒體插入到他們的頁面中。本章介紹了可用的選項。

 

RTF 編輯器

RTF 欄位可讓內容作者將格式化的內容插入其頁面。內容作者使用 RTF 編輯器來建立和編輯此格式化的內容。本節涵蓋的功能視頻雲網站核心媒體框架添加到富文本編輯器。

嵌入媒體

媒體可以直接嵌入到富文本字段中。

  1. 在 RTF 編輯器中，按一下「嵌入媒體」按鈕。
  2. 使用搜尋畫面來尋找您要插入的媒體。按一下代表您要插入之媒體的列。按一下「下一步 >」按鈕。
  3. 視您選取的媒體類型而定，您會看到其他選項。例如，如果您選取視訊，則必須指定要用來顯示視訊的播放程式，以及播放程式的尺寸。按一下「 完成」 。
  4. 媒體內嵌在 RTF 欄位中。

嵌入媒體連結

可以在 RTF 欄位中插入媒體的連結。

  1. 在 RTF 編輯器中，按一下「內嵌媒體連結」按鈕。
  2. 使用搜尋畫面來尋找您要插入的媒體。按一下代表您要插入之媒體的列。按一下「下一步 >」按鈕。
  3. 輸入「連結標題」的值。這是將出現在媒體鏈接的文本。
    此外，視您選取的媒體類型而定，您會看到其他選項。例如，如果您選取視訊，則必須指定要用來顯示視訊的播放程式，以及播放程式的尺寸。按一下「 完成」 。
  4. 連結內嵌在 RTF 欄位中。

彩現

彩現允許將內容放置在頁面上的特定位置。視頻雲網站核心媒體框架包括一個渲染，允許內容作者在頁面中插入媒體。

嵌入媒體顯示

  1. 使用「頁面編輯器」將呈現新增至預留位置。
  2. 選取「內嵌媒體」顯示。
  3. 彩現已加入至預留位置。現在點擊嵌入媒體元素按鈕。
  4. 使用搜尋畫面來尋找您要插入的媒體。按一下代表您要插入之媒體的列。按一下「下一步 >」按鈕。
  5. 視您選取的媒體類型而定，您會看到其他選項。例如，如果您選取視訊，則必須指定要用來顯示視訊的播放程式，以及播放程式的尺寸。按一下「 完成」 。
  6. 已選取媒體進行彩現。

之間的同步視頻雲網站核心和視頻雲

本節介紹如何保持視頻和播放列表之間的同步視頻雲網站核心用於以下場景：

  • 從布萊特灣刪除影片和播放清單時
  • 添加視頻和播放列表時視頻雲網站核心
  • 刪除視頻和播放列表時視頻雲網站核心

從 Brightcove 導入到視頻雲網站核心之前被覆蓋這裡 .

匯出視訊

之內視頻雲網站核心 , Brightcove 媒體使用視頻雲網站核心在媒體導入過程中創建的項目。既然他們是視頻雲網站核心項目，用戶可以更改這些項目。媒體導出過程負責從視頻雲網站核心到布賴特科夫。

自動匯出媒體

當一個視頻雲網站核心與 Brightcove 媒體對應的項目在視頻雲網站核心客戶端，這些更改將被推送到 Brightcove。

即時匯出

在創建新媒體項目時視頻雲網站核心，默認情況下執行實時導出。這表示會立即執行匯出工作。

防止自動匯出

您可能不希望在視頻雲網站核心出口到 Brightcove。您可能希望連接來自視頻雲網站核心到 Brightcove 為只讀，其中 Brightcove 更改導入到視頻雲網站核心但是哪裡視頻雲網站核心更改不會導出到 Brightcove。

為了防止在視頻雲網站核心從導出到 Brightcove，清除以下設置的值：

設定檔： Video Cloud Sitecore .MediaFramework.config
設定名稱： Video Cloud Sitecore .MediaFramework.ExportDatabases
 

注意：此設置適用於視頻雲網站核心媒體框架，而不僅僅是 Brightcove Video Connect 插件。變更此設定將會套用至所有連接器和所有帳戶。

資料清除

清理過程用於保持視頻雲網站核心和 Brightcove 數據同步。

視頻雲網站核心清理

視頻雲網站核心清理是在 Brightcove 中刪除媒體時運行的過程，以及相應的視頻雲網站核心項必須刪除。由於 Brightcove 對視頻雲網站核心 , Brightcove 無法通知視頻雲網站核心該媒體已被刪除。所以，視頻雲網站核心必須積極主動。

手動清除

一種視頻雲網站核心內容作者或管理員可以通過單擊內容編輯器中的以下按鈕手動觸發清理過程：

自動清除

Brightcove 視訊連線外掛程式包含定期自動執行清理程序的代理程式。代理程式定義在下列位置：

設定檔： Video Cloud Sitecore .MediaFramework.Services.Brightcove.config
代理程式名稱： MediaFramework_Cleanup_Brightcove

布萊特灣清理

Brightcove 清理是在刪除媒體項目時運行的過程視頻雲網站核心並且必須刪除或禁用 Brightcove 中的相應媒體。由於 Brightcove 知道媒體項目已被刪除，視頻雲網站核心能夠通知 Brightcove 應刪除或禁用 Brightcove 中的相應媒體。

當媒體項目在視頻雲網站核心客戶端，在導出隊列中創建一個條目。此程序與儲存媒體項目時相同。

播放事件分析

這視頻雲網站核心媒體框架允許記錄視頻播放事件以用於視頻雲網站核心數字營銷系統 (DMS)。本節涵蓋可用的選項。

回放事件

播放事件用於記錄 DMS 頁面事件作為視訊播放器動作的結果。例如，當視頻播放開始時，可以寫入 DMS 頁面事件。

預設回放事件

視頻雲網站核心媒體框架帶有以下播放事件：

  • 開始播放
  • 播放完成
  • 播放錯誤

設定帳號的播放事件

依預設，會錄製回放開始和回放已完成的事件。此功能會在 [Brightcove 帳號] 項目下的 [設定] 項目上啟用。

「設定」項目有一個名為「播放事件規則」的欄位。定義了一個規則，告訴媒體框架記錄在視頻和播放列表下指定的事件。

停用帳號的播放事件

您可以針對特定帳號停用所有播放事件。

  1. 在內容編輯器中，導覽至 [媒體庫] > [媒體架構] > [帳戶] 下的帳戶。
  2. 選取「設定」項目。
  3. 「播放事件規則」(Playback Event Rules) 欄位描述必須符合的條件，才能註冊播放事件。默認情況下，條件是「哪裡真」，意思是「總是」。按一下「編輯規則」按鈕以變更條件，或移除現有規則。

指定媒體的播放事件

每個播放清單和視訊項目都有名為「事件」的欄位。此欄位決定要處理指定項目的播放事件。

注意：只有在 Brightcove 帳戶項目下的 [設定] 項目的 [播放事件規則] 欄位設定為 [新增媒體項目事件] 動作時，才會記錄這些事件。請參閱第 3.1.2 節。

建立新的播放事件

您可以新增播放事件，以擷取更多有關視訊播放的詳細資訊。

  1. 在內容編輯器中，導航到系統 > 模塊 > 媒體框架 > 播放事件
  2. 使用範本播放事件新增項目
  3. 對於頁面事件字段，請選擇播放已更改
  4. 在「參數」(Parameter) 欄位中，您可以輸入下列其中一種格式的值：
    1. 數字-視頻播放這個秒數後，頁面事件被記錄。有效值的範例為 15
    2. 百分比-播放視頻後，會記錄頁面事件。有效值的範例為 50%
  5. 儲存項目
  6. 發佈項目
  7. 現在，您可以將新的頁面事件添加到您的播放列表和視頻

注意：請記得將新的播放事件指派至播放清單和影片。否則，事件將不會被記錄。
